PHASE 1 — FOUNDATION
================================================================================

[P1-001] [x] Project scaffolding and configuration system
    - Python package structure
    - YAML settings file with all configurable parameters
    - Config loader with defaults and validation
    - Seeded random number generation for reproducibility

[P1-002] [x] Arena and map system
    - Rectangular map with configurable size (min/max bounds)
    - Terrain types: open ground, walls (impassable), mud (slow), road (fast)
    - Default map presets (selectable via settings)
    - Map rotation: generations cycle through preset list to prevent overfitting
    - Map loader/generator

[P1-003] [x] Tank model
    - Three tank types: light, medium, heavy
    - Properties per type (all configurable):
      * HP, armor, speed, acceleration, turn rate
      * Turret rotation speed, reload time, damage per shot
      * Ammo capacity, visibility range
    - Tank state machine: moving, firing, repairing, destroyed
    - Self-repair mechanic: tank stops, becomes vulnerable, restores HP
      over configurable time period
    - Repair is NOT interruptible — once started, must complete or tank
      is destroyed (sitting duck). Strategy decides whether to risk it.

[P1-004] [x] Physics and movement
    - 2D position, velocity, heading
    - Acceleration/deceleration model
    - Terrain speed modifiers
    - Tank-tank collision (no damage, just blocking)
    - Tank-wall collision
    - Turret rotation independent of hull

[P1-005] [x] Combat system
    - Projectile model (speed, range, damage — per tank type)
    - Configurable shell speed per tank type (ballistic leading required)
    - Line-of-sight check (walls block shots)
    - Hit detection
    - Friendly fire enabled
    - Ammo tracking and depletion
    - Reload cooldown

[P1-006] [x] Fog of war
    - Per-tank visibility radius (configurable per type)
    - Team vision sharing (allies share what they see)
    - Terrain blocking line of sight (walls)
    - Sensor data: enemy position, angle, distance, turret direction
    - No visibility of enemy HP

[P1-007] [x] Match system
    - Variable team sizes (1–10), configurable
    - Free-for-all mode
    - Win condition: last team standing
    - Time limit with tiebreaker (most damage dealt wins)
    - Best-of-N matches (N configurable)
    - Match result recording
    - Spawn positions: opposite sides of map, ensuring no terrain blockage

[P1-008] [x] Seed strategies and generation 0
    - Configurable initial population: fully random trees OR
    - Seed strategies: hand-crafted starting strategies loaded from file
    - Mix mode: some seeds + random fill
    - Seed strategies allow reproducing specific scenarios or
      bootstrapping evolution from known-good starting points

================================================================================
PHASE 2 — STRATEGY & EVOLUTION
================================================================================

[P2-001] [x] Strategy representation — behavior trees
    - Node types: selector, sequence, condition, action
    - Condition nodes: enemy_visible, health_below, ammo_below,
      ally_nearby, under_fire, turret_aimed_at_me
    - Action nodes: move_toward, move_away, patrol, seek_cover,
      aim_at_nearest, aim_at_weakest_visible, fire, repair,
      signal_allies, move_to_signal
    - Numeric parameters on nodes (thresholds, weights, distances)
    - Serialization/deserialization of behavior trees

[P2-002] [x] Tank communication channel
    - Simple signal types (e.g., "enemy here", "help", "regroup")
    - Signal has position and type
    - Allies within visibility range receive signals
    - Strategy can react to received signals

[P2-003] [x] Evolution engine — core
    - Population of strategies per team (default: 100)
    - Tournament selection
    - Parameter mutation (gaussian perturbation, configurable sigma)
    - Crossover (subtree swap between two parent trees)
    - Elitism (top-K preserved unchanged, K configurable)
    - Generation counter (default: 100 generations)
    - All evolution parameters configurable in settings

[P2-007] [x] Team composition evolution
    - Team composition (which tank types to field) is part of the genome
    - Strategy genome = behavior tree + composition vector
    - Composition vector: how many light/medium/heavy (sums to team_size)
    - Composition mutates alongside behavior: swap a light for a heavy, etc.
    - Different compositions compete — evolution discovers optimal mixes
    - Allows emergent archetypes: "all-light swarm", "heavy core + scouts", etc.

[P2-004] [x] Structural mutations
    - Insert random node into behavior tree
    - Delete node (promote child)
    - Swap subtrees between positions
    - Replace node type
    - Mutation probability configurable per operation type

[P2-005] [x] Independent team evolution
    - Each team maintains its own population pool
    - Teams evolve in parallel but compete against each other
    - Co-evolutionary dynamics (arms race tracking)

[P2-006] [x] Fitness evaluation
    - Fitness function components (weighted, configurable):
      * Win/loss outcome
      * Damage dealt
      * Damage taken (negative)
      * Survival time
      * Ammo efficiency
      * Team coordination score
    - Each strategy plays 3–5 opponents per generation (configurable)
    - Fitness averaged across all matches for robustness
